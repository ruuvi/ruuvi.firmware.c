◆ app_comms_ble_init()

rd_status_t app_comms_ble_init ( const bool  secure)

Enable Bluetooth on device.

After calling this function GATT server is initialized and ready to accept connection if GATT is enabled in application. Advertisements can be sent.

If ble comms are initialized as secure, DFU service is disabled and serial number is not readable over GATT.

If ble comms are initialized as unsecure, DFU service is enabled and serial number is readable over GATT.

Ble must be uninitialized to re-enter secure mode or vice versa.

Parameters
[in]secureTrue to not enable DFU service and serial number read over GATT. False to enable DFU service and serial number read over GATT.

rd_status_t app_comms_blocking_send ( const ri_comm_xfer_fp_t  reply_fp,
ri_comm_message_t *const  msg 
)

Blocking send message function.

Calls reply_fp with given data, and if reply_fp returns ERR_NO_MEM yields and retries. Has optional timeout. Function will return once message has been queued to driver buffer, not necessarily sent.

Parameters
[in]reply_fpFunction pointer to use to send the data.
[in]msgMessage to send.
Return values
RD_SUCCESSMessage was queued to TX buffer.
RD_ERROR_TIMEOUTMessage was queued to TX buffer. Error code from reply_fp is also included, e.g. RD_ERROR_TIMEOUT | RD_ERROR_INVALID_STATE
Note
Timeout requires RTC and some process which brings thread out of yield.

◆ app_comms_configure_next_disable()

rd_status_t app_comms_configure_next_disable ( void  )

Kick current BLE connection (if any) and disable security-sensitive configuration options.

After calling this function:

  • DFU service is disabled
  • Device ID is not readable through DIS/Serial Number characteristic.
  • Configuration commands are not allowed.

This also calls app_led to indicate configuration is disabled.

Return values
RD_SUCCESSif configuration mode was entered.
RD_ERROR_INVALID_STATEif communications are not initialized.

◆ app_comms_configure_next_enable()

rd_status_t app_comms_configure_next_enable ( void  )

APP_COMMS_BIDIR_ENABLED.

Enable security-sensitive configuration options.

After calling this function:

  • DFU service is enabled which allows firmware updates.
  • Device ID is readable through DIS/Serial Number characteristic.
  • Configuration commands are allowed.

The configuration is enabled for the next communication connection and is disabled after disconnection or once timeout is triggered. This kicks current Bluetooth connection (if any).

This also calls app_led to indicate configuration is enabled.

Return values
RD_SUCCESSif configuration mode was entered.
RD_ERROR_INVALID_STATEif communications are not initialized.

◆ app_comms_init()

rd_status_t app_comms_init ( const bool  secure)

Initialize communication methods supported by board.

After calling this function, NFC is ready to provide device information, advertisement data can be sent and advertisements are connectable for enabling GATT connection.

Use ruuvi task functions, such as rt_gatt_send_asynchronous to send data out.

Parameters
[in]secureIf true, GATT won't expose DFU service and device ID. If false, DFU service is open and device ID can be read through GATT.
Return values
RD_SUCCESSon success.
Returns
error code from stack on error.
